SUPPORT DE COURS : ALGEBRE RELATIONNEL & LANGAGE SQL

Partie I: Algèbre relationnel

1. Introduction à l'Algèbre Relationnelle

L'algèbre relationnelle est un langage formel utilisé pour manipuler et interroger des bases de données relationnelles. Elle repose sur des opérations permettant de traiter des relations (tables) et d'obtenir des résultats sous forme de nouvelles relations. Elle constitue le fondement théorique des langages de requêtes comme SQL.

2. Les Opérations Fondamentales

L'algèbre relationnelle comprend plusieurs opérations fondamentales permettant de filtrer et de transformer les données stockées dans les relations.

a. Opération de Sélection (σ)

La sélection permet d'extraire les lignes (ou tuples) d'une relation qui vérifient une condition donnée.

Notation :
σcondition(R)

Où R est une relation et condition est un critère de filtrage.

Exemple :

Si nous avons une table Employés (Nom, Age, Salaire), et nous voulons sélectionner les employés ayant un salaire supérieur à 3000, nous écrivons : 

σSalaire>3000(Employes)

Résultat de la requête (Tableau Résultant) :

Nom Age Salaire
Alice 30 3500
Charles 45 4200

Cette requête retourne tous les employés de la table "Employes" dont le salaire est strictement supérieur à 3000.

b. Opération de Projection (π)

La projection permet de sélectionner certaines colonnes d'une relation.

Notation :
πattributs(R)

Où attributs sont les colonnes à conserver.

Exemple :

πNom, Salaire(Employes)

Renvoie une nouvelle relation ne contenant que les colonnes Nom et Salaire.

Résultat de la requête (Tableau Résultant) :

Nom Salaire
Alice 3500
Bob 2800
Charles 4200

Cette requête retourne une nouvelle table contenant uniquement les noms et salaires des employés.

c. Opération de Jointure (⋈)

La jointure permet de combiner des relations sur la base d'un attribut commun.

Notation :
R ⋈condition S

Où R et S sont deux relations et condition est le critère de jointure.

Exemple :

Si nous avons une table Employés (Id, Nom, #IdDept) et une table Départements (IdDept, NomDept), une jointure sur IdDept permet d'obtenir les noms des employés avec leur département: 

Employes ⋈Employes.IdDept=Departements.IdDept Departements

Résultat de la requête (Tableau Résultant) :

Employes.Id Employes.Nom Employes.#IdDept Departements.IdDept Departements.NomDept
1 Alice 101 101 Ventes
2 Bob 102 102 Marketing

Cette jointure combine les informations des employés avec les détails de leurs départements respectifs en se basant sur l'identifiant du département.

3. Opérateurs Algébriques

a. Union (∪)

L'union combine deux relations ayant les mêmes attributs et élimine les doublons.

Notation :
R ∪ S

Exemple :

Si R et S sont deux relations contenant des employés, alors R ∪ S renvoie une relation avec tous les employés des deux tables, sans doublons.

L'opération d'union combine les employés des deux départements sans inclure de doublons.

b. Intersection (∩)

L'intersection retourne les éléments communs aux deux relations.

Notation :
R ∩ S

Exemple :

Si R et S contiennent des employés, R ∩ S renverra les employés présents dans les deux relations.

La différence retourne les éléments qui sont dans la première relation mais pas dans la seconde.

Notation :

Le produit cartésien combine chaque ligne de la première relation avec chaque ligne de la seconde, créant ainsi une relation plus large.

Partie II: Langage SQL

1. Introduction au Langage SQL

SQL (Structured Query Language) est un langage informatique normalisé servant à exploiter des bases de données relationnelles. Il se divise en plusieurs catégories : le langage de définition des données (DDL), le langage de manipulation des données (DML) et le langage de contrôle des données (DCL).

Les instructions de manipulation des métadonnées (structure de la base) commencent avec les mots clés CREATE, ALTER ou DROP. Les instructions de manipulation du contenu de la base de données commencent par les mots clés SELECT, UPDATE, INSERT ou DELETE. Enfin, GRANT et REVOKE sont utilisés pour les autorisations.

Quelques règles d'écritures en SQL :

  • Un nom d'objet (table, colonne, etc.) ne doit pas dépasser 128 caractères, commencer par une lettre, et être composé de lettres, chiffres ou tirets bas (_).
  • Ne pas être un mot réservé en SQL.
  • Pas de caractères accentués.
  • Les noms d'objets sont insensibles à la casse.
  • SQL ne fait pas la différence entre les minuscules et les majuscules.
  • Les commentaires de code sont entre double tirets (--).
  • Ne pas oublier le point-virgule (;) à la fin de chaque instruction pour déclencher son exécution.

1. SQL langage de contrôle des données (DCL)

Le DCL gère la sécurité et la cohérence de la base de données en contrôlant l'accès aux informations. Il inclut la gestion des utilisateurs, des privilèges (système et objet) et des rôles.

a. La gestion des utilisateurs

Un utilisateur est identifié par son nom et un mot de passe et doit posséder des privilèges pour se connecter et accéder aux objets de la base.

Création d'un utilisateur :

CREATE USER <nom_id> IDENTIFIED BY <password>;

Exemple : Créer un utilisateur 'mon_utilisateur' avec le mot de passe 'password123'.

CREATE USER mon_utilisateur IDENTIFIED BY password123;

Cette commande crée un nouvel utilisateur dans la base de données Oracle. Notez qu'après la création, l'utilisateur n'a aucun privilège par défaut.

Modification de mot de passe :

ALTER USER <nom_id> IDENTIFIED BY <new_password> REPLACE <old_password>;

Exemple : Modifier le mot de passe de 'mon_utilisateur' de 'password123' à 'new_password456'.

ALTER USER mon_utilisateur IDENTIFIED BY new_password456 REPLACE password123;

Cette commande modifie le mot de passe de l'utilisateur spécifié.

Suppression d'un utilisateur :

DROP USER <nom_id> [CASCADE];

CASCADE supprime également tous les objets appartenant à l'utilisateur.

Exemple : Supprimer l'utilisateur 'mon_utilisateur' et tous ses objets.

DROP USER mon_utilisateur CASCADE;

Cette commande supprime l'utilisateur et toutes ses tables, vues, etc.

b. La gestion des privilèges

Un privilège est un droit d'exécuter une instruction SQL (privilège système) ou d'accéder à un objet (privilège objet).

Privilèges systèmes :

GRANT privilègesystème | ALL PRIVILEGES TO utilisateur | public [IDENTIFIED BY mot_passe] [WITH ADMIN OPTION];
  • ALL PRIVILEGES: tous les privilèges système.
  • PUBLIC: attribue le privilège à tous les utilisateurs.
  • WITH ADMIN OPTION: permet au bénéficiaire de transmettre les privilèges.

Exemple : Accorder le droit de connexion et de création de tables à 'Hicham'.

GRANT CREATE SESSION, CREATE TABLE TO Hicham;

Hicham peut désormais se connecter à la base et créer des tables dans son schéma.

Exemple : Accorder le droit de créer des tables à 'Hicham' avec la possibilité de transmettre ce privilège.

GRANT CREATE TABLE TO Hicham WITH ADMIN OPTION;

Hicham peut créer des tables et également accorder ce privilège à d'autres utilisateurs.

Retrait de privilège système :

REVOKE privilègesystème | ALL PRIVILEGES FROM utilisateur | PUBLIC;

Exemple : Retirer le droit de connexion à 'Hicham' et 'Ahmed'.

REVOKE CREATE SESSION FROM Hicham, Ahmed;

Hicham et Ahmed ne peuvent plus se connecter à la base de données.

Privilèges objets :

Ces privilèges sont relatifs aux données et aux actions sur des objets spécifiques (table, vue, séquence, procédure).

Attribution de privilèges objets :

GRANT privilègeobjet | ALL PRIVILEGES [(colonne 1, colonne 2)] ON SCHEMA.nomobjet TO utilisateur | PUBLIC [WITH GRANT OPTION];

Exemple : Accorder les droits de lecture, insertion, modification et suppression sur la table ETUDIANTS du schéma SMI à 'utilisateur1'.

GRANT SELECT, INSERT, UPDATE, DELETE ON SMI.ETUDIANTS TO utilisateur1;

L'utilisateur1 peut effectuer toutes les opérations de manipulation de données sur la table ETUDIANTS.

Exemple : Accorder le droit de modifier uniquement les colonnes 'note' et 'moy' de la table ETUDIANTS du schéma SMI à 'utilisateur1'.

GRANT UPDATE (note, moy) ON SMI.ETUDIANTS TO utilisateur1;

L'utilisateur1 est restreint à modifier uniquement les colonnes spécifiées.

Retrait de privilège objet :

REVOKE [GRANT OPTION FOR] Priv1, Priv2 ... ON objet FROM utilisateur, utilisateur2... | PUBLIC [RESTRICT | CASCADE];
  • GRANT OPTION FOR: retrait uniquement du droit de transfert.
  • CASCADE: retrait des droits accordés à des tiers par le bénéficiaire.
  • RESTRICT: refuse le retrait en cascade.

Exemple : Retirer les droits de modification (nom, age) et de lecture sur la table Pilote de 'Hicham' à 'Ahmed'.

REVOKE UPDATE(nom, age), SELECT ON Hicham.Pilote FROM Ahmed;

Ahmed ne peut plus lire ni modifier la table Pilote.

c. La gestion des rôles

Un rôle est un regroupement de privilèges (système ou objet) qui peut être accordé à un ou plusieurs utilisateurs. Cela facilite la gestion des privilèges.

Création/suppression de rôles :

CREATE ROLE nom_role;
DROP ROLE nom_role;

Exemple : Créer un rôle 'comptabilite'.

CREATE ROLE comptabilite;

Un nouveau rôle 'comptabilite' est créé.

Association d'un rôle à un utilisateur :

GRANT nom_role TO utilisateur;

Exemple : Accorder le rôle 'comptabilite' à 'utilisateur_compta'.

GRANT comptabilite TO utilisateur_compta;

L'utilisateur 'utilisateur_compta' hérite de tous les privilèges définis pour le rôle 'comptabilite'.

Exercice d'application : SQL (DCL)

Une entreprise utilise une base de données Oracle pour gérer ses employés. On a deux utilisateurs :

  • admin_hr: Responsable des ressources humaines, il doit gérer les employés.
  • user_hr: Un employé standard qui peut consulter les informations mais ne doit pas les modifier.

L'entreprise possède une table employes contenant les informations des employés.

Objectifs de l'exercice :

  1. Créer les utilisateurs admin_hr et user_hr.
  2. Attribuer les privilèges appropriés.
  3. Tester l'accès aux données.

Correction :

1. Création des utilisateurs :

CREATE USER admin_hr IDENTIFIED BY admin_password;
CREATE USER user_hr IDENTIFIED BY user_password;

Ces commandes créent les deux utilisateurs avec leurs mots de passe respectifs.

2. Attribution des privilèges :

Pour admin_hr (gestion complète de la table employes) :

GRANT CREATE SESSION TO admin_hr;
GRANT SELECT, INSERT, UPDATE, DELETE ON votre_schema.employes TO admin_hr;

Pour user_hr (lecture seule de la table employes) :

GRANT CREATE SESSION TO user_hr;
GRANT SELECT ON votre_schema.employes TO user_hr;

Les privilèges nécessaires sont accordés à chaque utilisateur en fonction de son rôle. Remplacez votre_schema par le nom du schéma où se trouve la table employes.

Se connecter en tant que user_hr et essayer de lire les données :

-- Essayer d'insérer (devrait échouer) :

Ces étapes permettent de vérifier que les privilèges ont été correctement appliqués.

2. SQL langage de définition de données (DDL)

Le DDL permet de créer, modifier et supprimer la structure des objets de la base de données (tables, vues, index).

a. Création de table

Pour créer une table, on utilise l'instruction CREATE TABLE en spécifiant le nom de la table, les noms des colonnes, leurs types de données et leurs contraintes. 

CREATE TABLE <nom de la table> (
    <nom de colonne> <type> <contrainte>,
    <nom de colonne> <type> <contrainte>,
    <contrainte>,
    ...
);

Exemple : Créer les tables CLIENT et COMMANDE.

CREATE TABLE Client (
    NoClient Integer,
    NomClient Varchar(25),
    TypeClient VarChar(15)
);

CREATE TABLE Commande (
    NoCommande Integer,
    DateCommande Date,
    DateLivraison Date,
    NoClient Integer
);

Ces commandes créent les structures des tables CLIENT et COMMANDE.

b. Définition des contraintes d'intégrité

Les contraintes garantissent la cohérence des données. Les mots clés courants sont NOT NULL, UNIQUE, CHECK, PRIMARY KEY, FOREIGN KEY et DEFAULT.

Exemple de création de table avec contraintes :

CREATE TABLE clients(
    Nom char(30) NOT NULL,
    Prenom char(30) NOT NULL,
    Age integer CHECK (Age < 100),
    Email char(50) NOT NULL, CHECK (Email LIKE '%@%')
);

Exemple : Créer la table Client avec des contraintes de clé primaire et de vérification.

CREATE TABLE Client (
    NoClient Integer,
    NomClient VARCHAR(25) NOT NULL,
    TypeClient VARCHAR(15) NOT NULL,
    CONSTRAINT PK_Client PRIMARY KEY (NoClient),
    CONSTRAINT CK_Client CHECK (TypeClient = 'PARTICULIER ' OR TypeClient = 'PROFESSIONNEL ')
);

Cette commande crée la table Client avec une clé primaire sur NoClient et une contrainte qui limite les valeurs de TypeClient à 'PARTICULIER' ou 'PROFESSIONNEL'.

Règles à respecter pour garantir l'intégrité référentielle :

updateCorrecte si l'instruction ne laisse pas d'enregistrements dans la table fils ayant une clé étrangère non référencéeCorrecte si la nouvelle clé étrangère référence un enregistrement père existant
deleteCorrecte si aucun enregistrement de la table fils ne référence le ou les enregistrements détruitsCorrecte sans condition

L'option ON DELETE CASCADE permet de supprimer une ligne d'une table ainsi que toutes les lignes liées dans une autre table.

c. Modification de la structure d'une table

Ajout de colonnes ou contraintes :

ALTER TABLE <nom de la table> ADD (<nom de colonne> <type>[, <contrainte>]...);
ALTER TABLE <nom de la table> ADD (<contrainte>, <contrainte>...);

Exemple : Ajouter la colonne 'CodePostal' à la table 'Client'.

ALTER TABLE Client ADD (CodePostal VARCHAR(10));

Cette commande ajoute une nouvelle colonne à la table 'Client'.

Modification de colonnes :

ALTER TABLE <nom de la table> MODIFY ([<nom de colonne> <nouveau type>, <nom de colonne> <nouveau type>,...);

Exemple : Modifier le type de 'NomClient' dans la table 'Client'.

ALTER TABLE Client MODIFY (NomClient VARCHAR(50));

Cette commande modifie la taille maximale de la colonne 'NomClient'.

d. Consultation de la structure d'une base

DESCRIBE <nom de la table>;

Exemple : Décrire la structure de la table 'Client'.

DESCRIBE Client;

Cette commande affiche les détails de la structure de la table, y compris les noms des colonnes, leurs types et si elles sont nullables.

e. Destruction de table

DROP TABLE <nom de la table>;

Exemple : Supprimer la table 'Commande'.

DROP TABLE Commande;

Cette commande supprime la table 'Commande' de la base de données.

3. SQL langage de manipulation de données (DML)

Le DML permet de manipuler les données dans les tables (insertion, modification, suppression et interrogation).

a. Insertion de données (INSERT)

Pour insérer une ou plusieurs lignes dans une table. 

INSERT INTO <nom de table> (colonne, ...) VALUES (valeur, ...);
INSERT INTO <nom de table> (colonne, ...) SELECT ...;

Exemple : Insérer un nouveau client dans la table 'clients'.

INSERT INTO clients (num_client, nom_client, prenom_client) VALUES (10, 'morabit', 'ahmed');

Cette commande ajoute une nouvelle ligne à la table 'clients'.

Exemple : Insérer un client avec un numéro auto-incrémenté via une séquence.

CREATE SEQUENCE sq_clients
MINVALUE 1
MAXVALUE 99
START WITH 1
INCREMENT BY 1;

INSERT INTO clients (Num, Nom, Prenom) VALUES (sq_clients.NEXTVAL, 'morabit', 'ahmed');

La séquence sq_clients est créée et utilisée pour générer automatiquement l'ID du client.

b. Modification de données (UPDATE)

Pour modifier la valeur d'une ou plusieurs colonnes, d'une ou plusieurs lignes. 

UPDATE <nom de table> SET colonne = valeur, ... [WHERE condition];
UPDATE <nom de table> SET colonne = SELECT ...;

Exemple : Augmenter le prix unitaire des articles de 15.2449%.

UPDATE articles SET prix_unitaire=prix_unitaire*0.152449;

Cette commande met à jour le prix de tous les articles.

Exemple : Doubler le prix unitaire des articles de couleur 'rouge'.

  • FLOOR(n): partie entière de n.

  • MOD(n, m): n modulo m.
  • SIGN(n): 1 si n>0, 0 si n=0, -1 si n<0.
  • SQRT(n): racine carrée de n.
  • GREATEST(n1, n2,...): la plus grande valeur de la suite.
  • LEAST(n1,n2,...): la plus petite valeur de la liste.
  • NVL(n1, n2): substitue n2 à n1 si n1 est nul.
  • LENGTH(ch): longueur de la chaîne.
  • SUBSTR(ch, pos [, long]): extraction d'une sous-chaîne.
  • INSTR(ch, ssch [, pos [, n]]): position de la sous-chaîne dans la chaîne.
  • UPPER(ch): mise en majuscules.
  • LOWER(ch): mise en minuscules.

Calculs verticaux (fonctions agrégatives)

Les fonctions agrégatives s'appliquent à un ensemble de valeurs d'un attribut (numérique sauf COUNT). Le résultat est évalué une seule fois pour tous les tuples.

  • SUM: somme.
  • AVG: moyenne arithmétique.
  • COUNT: nombre ou cardinalité.
  • MAX: valeur maximale.
  • MIN: valeur minimale.
  • STDDEV: écart type.
  • VARIANCE: variance.

Exemple : Quel est le salaire moyen des pilotes Marseillais ?

SELECT AVG(SALAIRE)
FROM PILOTE
WHERE ADRESSE = 'MARSEILLE';
AVG(SALAIRE)
40000

Exemple : Trouver le nombre de vols au départ de Marseille.

SELECT COUNT(NUMVOL)
FROM VOL
WHERE VILLE_DEP = 'MARSEILLE';
COUNT(NUMVOL)
2

COUNT(*) peut aussi être utilisé.

Expression des jointures sous forme prédicative

Les jointures extraient des données de plusieurs tables. La clause FROM contient les noms de toutes les tables, et WHERE exprime le critère de jointure. Si n tables sont dans FROM, il faut au moins (n-1) opérations de jointure pour éviter un produit cartésien non désiré. 

SELECT colonne,... FROM table1, table2... WHERE <condition>;

FROM Employe

FROM Employe

SELECT nom_emp
FROM Employe
